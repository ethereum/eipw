prevent-urls-no-backticks

This rule prevents putting URLs in backticks.

Examples

error[markdown-prevent-url]: URL in backticks or not from allowed domain found
  |
5 | Check this URL: `http://example.com/page?query=foo`
  |

Explanation

prevent-urls-no-backticks flags URLs that are in backticks, which can make them less readable or clickable. It is important to avoid URLs in backticks to ensure content integrity.

Backticks are generally used for code snippets or inline code, not for wrapping URLs. Including them can cause confusion and disrupt the readability or functionality of the URLs.

This rule ensures that URLs are formatted correctly, and that only specific domains (if configured) are allowed to be used, adding an additional layer of security and ensuring consistent documentation.
