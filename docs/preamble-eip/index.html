preamble-eip

The eip header is a non-negative integer.

Examples

error[preamble-eip]: preamble header `eip` must be an unsigned integer
 --> input.md:2:5
  |
2 | eip: 1abc
  |     ^^^^^ not a non-negative integer
  |

Explanation

preamble-eip makes sure the eip front matter (or preamble) field is a non-negative integer.

Each proposal is identified with a number. The eip field holds that number.

Before an EIP Editor assigns a number, this field may contain eip: <to be assigned>. The linter reports that placeholder with a separate message instead of treating it like a malformed number.
